# 长期记忆

## 技术要点

### 核心技术
- 每次请求都重新连接 conn（不复用连接）
- request/response 不需要 header，保持简化
- 处理器返回数据时自动装箱，send 函数返回时自动拆箱
- 在 `conn.on('data')` 获得响应后，拆包之前要校验返回状态是否正确
- **断线重连**：网络断开或连接失败时，每秒自动重连
- Peer ID 由本地生成（使用 `crypto.randomUUID()`），不依赖服务器分配
- **调试模式**：`isDebug=true` 时打印事件日志，格式为 `{对象} {事件名} {事件变量}`
- **TTL 机制**：路由转发消息时使用 TTL（Time To Live）防止死循环，默认 128 跳

### 语音/视频通话技术要点
- 使用 PeerJS 的 MediaConnection（`peer.call()` 和 `peer.on('call')`）
- 通过 `navigator.mediaDevices.getUserMedia()` 获取麦克风/摄像头
- 静音通过 `MediaStreamTrack.enabled = false` 实现
- 视频开关通过 `MediaStreamTrack.enabled` 切换
- 同一时间只能有一个活跃通话（`activeCall` 检查）
- 通话超时 30 秒无应答自动挂断
- `destroy()` 时自动挂断活跃通话并清理监听器

### 代码分层解耦
- **问题**：单一文件 1300+ 行，难以维护和理解
- **解决**：拆分为多个模块
  - `PeerJsWrapper.ts` - 主类，生命周期管理
  - `CallSession.ts` - 通话会话
  - `Router.ts` - 路由管理
  - `MessageHandler.ts` - 消息处理
  - `RoutingDB.ts` - 路由表持久化
  - `constants.ts` - 常量定义
- **原则**：模块间通过回调函数（接口）通信，降低耦合，每个模块职责单一

### 开发规范
- **完整注释**：每个文件、类、方法都要有 JSDoc 注释
- **内容应包括**：
  - 文件/模块功能说明
  - 类用途说明
  - 公开方法的参数、返回值说明
  - 关键逻辑的流程说明
- **原则**：好代码不需要太多解释，但必要的文档能帮助后续维护

### 代码优化规范
- **常量管理**：所有魔法数字提取到 `src/constants.ts` 集中管理
- **常量命名**：使用大写下划线命名（如 `CONNECTION_TIMEOUT_MS`）
- **常量类型**：
  - 超时时间：`CONNECTION_TIMEOUT_MS` (30秒), `SEND_TIMEOUT_MS` (10秒), `RECONNECT_DELAY_MS` (1秒)
  - 容量限制：`MAX_ROUTING_ENTRIES` (50), `MAX_DIRECT_NODES` (5)
  - 周期任务：`ROUTE_CLEANUP_INTERVAL_MS` (60秒), `ROUTE_BROADCAST_INTERVAL_MS` (30秒)
  - 过期时间：`ROUTE_EXPIRE_AGE_MS` (5分钟)
  - TTL：`DEFAULT_TTL` (128跳)
- **代码复用**：提取重复逻辑为通用方法，减少代码冗余
- **错误处理**：统一错误处理模式，避免重复的 try-catch

## 用户喜好

- 使用 TypeScript 开发
- E2E 测试只需要测试 Chrome 浏览器
- 简洁的 API 设计，不需要复杂的 header 机制
- **测试自动化**：测试时如果发现服务未启动，AI 应该自己启动服务
- **CSS 简洁**：样式代码不要过于冗长，保持精简
- **README 简洁**：README.md 里不要写 `本地运行 Demo`，直接提供在线 demo 链接即可

## Vue3 Demo 技术要点

### 项目结构
- 使用 Vite + Vue3 构建
- 使用 `vite-plugin-singlefile` 插件打包为单 HTML 文件
- 所有 JS/CSS 内联到 HTML，可直接在浏览器打开运行

### 核心功能
- 文字聊天、图片/文件传输、语音/视频通话
- 二维码展示 Peer ID（使用 `qrcode` 库）
- 扫码连接（使用 `html5-qrcode` 库）
- 使用 PeerJS 官方公共信令服务器（`0.peerjs.com:443`）

### E2E 测试
- 测试辅助函数：通过 `window.connectToPeer()` 直接连接
- 测试覆盖：Peer ID 显示、连接建立、消息收发、图片传输、通话按钮
