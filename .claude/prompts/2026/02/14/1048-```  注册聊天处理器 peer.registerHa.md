```
      peer.registerHandler('/chat', async (data) => {
        const { text, from } = data;

        // 保存消息
        const msg = await saveMessage(from, text, false);

        // 确保发送者在联系人列表中
        if (!contacts.find(c => c.peerId === from)) {
          await saveContact(from);
          contacts.push({ peerId: from });
          renderContacts();
        }

        // 更新UI

          appendMessage(msg);
        }

        return { received: true };
      });
```

我感觉这种方式不太好，需要对端send的时候在data里放一个from。
其实，底层完全知道from，为什么不能由底层来告诉应用层from是什么呢？
应用层专注于业务数据对象即可。
from这个字段属于底层应该管的字段。

建议api改成这样
```
peer.registerHandler('/chat', async (from: string, data: obj) => {
```
from表示对方的peerId，data是业务数据对象。这样底层字段不会入侵业务数据对象。
