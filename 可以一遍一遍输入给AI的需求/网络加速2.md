send(peerId: string, path: string, data?: unknown): Promise<unknown> 优先进行直连，无法直连自动降级，根据路由表去做转发。由快到慢的选择下一跳进行发送，如果一个下一跳转发失败，就选下一个，直到路由表里所有节点都试过为止。

A节点路由表设计
```
目标节点 | 下一跳
B       | C(50ms) 、D(100ms)  // 代表通过 C、D 去转发，都能连通到目标。
```

路由表记录转发情况下的下一跳节点，可能有多个下一跳节点，记录这个节点，以及这个节点通往目标节点的时间毫秒数。下一跳可能是直连，也可能是转发。
直连失败，且路由表也是空的，那就执行路由发现逻辑，向我能直连的所有其他节点广播询问，广播问谁能和目标进行通信，能通信的就回复我能，并且告诉我有多快，我就填入路由表。然后再根据路由表进行转发。

路由表只要记录下一跳的节点即可，根据下一跳的节点信息，把消息发送给中转节点，由中转节点去想办法把消息发送到目标，能直连就直连，不能的话也是根据自身的路由表，他也要转发就接着转发。都失败了就返回失败消息给源节点。
这样的机制下，就可以实现多跳，A -> B -> C -> D ... 能接力下去，多少跳都行。

需要记住我能直连的所有节点，以及和这个节点的通信时间是多少毫秒。

本节点收到其他节点的广播询问时，判断能不能响应广播中的询问，能的话，要响应询问，并告诉他我和目标节点的最快通信时间。
