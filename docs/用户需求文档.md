# 用户需求文档

## 1. 项目概述

编写一个 Node.js 库工具，将浏览器端的 PeerJS 封装成简单易用的类似 HTTP 的 API。

## 2. 核心功能

### 2.1 API 封装要求

- 封装 API，底层使用 PeerJS
- 入参：对端设备的 peerId 和传递的 request 参数
- 返回值：Promise<Response>，即对端设备的返回值

### 2.2 连接管理

- 每次请求都重新连接 conn

### 2.3 简化设计

- request、response 不需要 header，可以简化

## 3. 目标环境

- 这个库不是为 Node.js 后端环境设计的
- 而是为浏览器设计的
- 未来仅提供给浏览器端使用

## 4. 技术要求

- 使用 TypeScript

## 5. 测试要求

- 需要 E2E 测试
